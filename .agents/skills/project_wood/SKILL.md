# Antigravity Skill: Project Wood (Local UX/CRO)

## Core Identity
- **Brand Name:** Project Wood
- **Vibe:** Family Trusted, Artisanal, Community-Focused [cite: 12, 13]
- **Primary Goal:** Lead generation for on-site wood deck consultations [cite: 126]
- **Target Device:** Mobile-First (Responsive Web) [cite: 55, 66]

## Design System Constraints
- **Color Palette:** 
  - Primary: Earthy Oak (#8B5A2B)
  - Secondary: Sun-Drenched Cedar (#C27E3A)
  - Accents: Hunter Green (#355E3B), Warm Cream (#FFFDD0) [cite: 24, 29]
- **Typography:** 
  - Headings: Serif (e.g., Playfair Display) for "Craftsmanship" feel [cite: 28, 133]
  - Body: Sans-Serif (e.g., Roboto) for mobile legibility [cite: 27]
- **UI Styling:** 
  - Large border-radius (16px+) for "friendly" rounded corners [cite: 28, 72]
  - Subtle drop shadows on cards to create depth/trust [cite: 85]

## Conversion Rate Optimization (CRO) Rules
1. **The "Call-to-Action" (CTA) Hierarchy:**
   - Primary: "Book Free On-Site Consultation" (High-contrast button) [cite: 18, 129]
   - Secondary: "Call Now" (Sticky header icon) [cite: 40, 71]
2. **Trust Sequencing:**
   - Always place "Trust Badges" (Licensed, Insured, Warranty) immediately below the Hero section [cite: 68, 71]
   - Testimonials must include "Neighbor" or "Local" keywords [cite: 137]
3. **Lead Form Simplicity:**
   - Max 4 fields for mobile: Name, Phone, Zip, and Project Type [cite: 29, 39]

## Stitch Prompting Instructions
- **Mode Selection:** Use `Standard Mode` for structural edits and Figma exports[cite: 58, 117]. Use `Experimental Mode` for high-fidelity visual overhauls[cite: 61, 98].
- **Iteration Protocol:** Focus on one section at a time (e.g., "Refine the Gallery grid") to avoid layout drift[cite: 16, 38].
- **Imagery:** Specify "Macro shots of wood grain" and "Lifestyle photos of families on decks" to balance product and trust[cite: 21, 142].

## Prohibited Patterns
- Avoid "Corporate" or "Industrial" aesthetics (No sharp 0px corners, no cold blues/greys)[cite: 13, 26].
- No "centered" app names; keep branding left-aligned for standard mobile UX[cite: 91, 137].
- Do not use dummy text for CTAs; use active, benefit-driven language[cite: 36, 135].
